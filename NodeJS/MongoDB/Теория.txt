Реляционная БД (например MySQL) - представляет из себя таблицы, в которых есть поля с различными типами данных, между этими
таблицами есть различные связи (од ин ко многим, многие ко многим). В реляционных базах данных есть язык запросов SQL.

MongoDB - Документированная база данных, хранит данные не в таблицах а в так называемых коллекциях. Коллекция - это
аналог таблици в реляционной БД. Каждая коллекция содержит в себе документы в формате *.json. Документ аналог строки из
реляц. БД, а поля (properties) - аналог столбцов.
Подводя итог: MongoDB состоит из баз данных, которые состоят из коллекций, которые состоят из документов в формате *.json.
Коллекции могут быть проиндексированы для ускорения доступа к ним, получение данных из MongoDB сводится к получению так
называемого курсора, который отдает эти данные по мере надобности.

MongoDB - бесструктурная БД. вставка происходит в разы быстрее. Чтение данных немного медленнее. Она асинхронная.
Можно создавать ограниченные коллекции db.createCollection с флагом, и задать размер коллекции. И если, например, размер
превышает 1мб, то старые записи удаляются - это потрясающе!
