Тэги - это ссылки на коммиты.
Единожды создав тэг на каком-либо коммите, он никогда не передвигается, в отличии от веток.
Удобно использовать для релизов


### git tag ###

git tag - вывести список тегов
git tag <tagname> <commit> - создать тэг на коммите
git tag -a -m <message> <tagname> <commit> - создать тэг с аннотацией (git show <tag> такого тэга покажет аннотацию и доп информацию о тэггере (авторе тега))
git tag -d <tagnames> - удалить теги
--флаги и фильтры
-l {mask} - показать теги, соответствующие маске, например, 'v1.1*'
git tag --contains <commit> - показать какие тэги содержит коммит
git tag -n{n} - вывести список тегов, с сообщениями на коммитах {n} строк, по умолчанию 1


### git describe и git archive ###

git describe <commit> - выводит тег коммита
git describe - примерный вывод 'v1.0.0-1-gd05d2f0' - это 'ближайший тег-на сколько коммитов мы впереди от ближайшего тега-g(commit)' - данное описание можно использовать с git archive
git archive -o /tmp/v1.0.0-1-gd05d2f0.zip HEAD - создает архив репозитория на момент нужного коммита (в данном случае HEAD)

P.S.:
эти команды работают с аннотированными тегами, если хотим использовать обычные теги, то надо найти эту опцию
