##### Копирование коммитов #####

git cherry-pick <commits> - скопировать коммиты (его diff) в текущую ветку (данная команда имеет состояние: в процессе или завершено)
--флаги git cherry-pick ...
-x - добавить в описание коммита информацию, откуда был скопирован коммит
--abort - отменить cherry-pick, при возникновении копирования коммитов
--continue - продолжит выполнение cherry-pick
--quit - сбросить состояние cherry-pick, сохранив удачные изменения в рабочей дериктории
--no-commit - добавить изменения в рабочую дерикторию и индекс, но не коммитить

git cherry <branch-1> <branch-2> - показать коммиты из ветки <branch-2> для которых есть копии в <branch-1> (минус - есть копия, плюс - нет копии)
--флаги git cherry ...
-v - добавить описание коммитов
